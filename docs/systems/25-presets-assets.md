# 25 — Presets & Assets

> Milestone: M3+ (brushes) · matures M10 · Status: Spec

## Purpose

PhotoEdit accumulates reusable assets: brush presets, tool presets, gradients,
patterns, swatches, styles (layer-effect presets), custom shapes, actions, LUTs,
workspaces, keyboard shortcuts, export presets, and Camera Raw presets. The preset
system gives all of them one model for storage, import/export, versioning,
thumbnails, search, categories, sync, and migration — so each asset type plugs in
rather than reinventing persistence.

## Requirements

**Functional**

- A generic, typed library per asset kind with categories and search.
- Preview thumbnails generated per preset.
- Import/export (single and pack), including reading common third-party formats
  where feasible (e.g. brush/gradient/swatch packs).
- Versioning and migration between app versions.
- Optional [cloud](28-cloud-account.md) sync of libraries.

**Non-functional**

- `pe_core` owns the preset model and serialization; the browser UI lives in the
  [app shell](24-ui-workspace.md).
- Loading is lazy (thumbnails first; payloads on use) so large libraries stay fast.

## Data model

```cpp
namespace pe {

struct PresetMeta {
    Uuid id;
    std::string name, category;
    std::vector<std::string> tags;
    int version = 1;
    ThumbnailRef thumbnail;
};

// T is the payload type: BrushSettings, Gradient, Pattern, Swatch, LayerStyle,
// Shape (Path), Action, Lut, ExportPreset, RawDevelopSettings, Workspace...
template <class T>
struct Preset { PresetMeta meta; T payload; };

template <class T>
class PresetLibrary {
public:
    std::span<const PresetMeta> list() const;          // thumbnails/metadata only
    const T& load(Uuid);                               // fault payload on demand
    Uuid add(std::string name, T payload);
    void remove(Uuid);
    void importPack(const Path&); void exportPack(std::span<const Uuid>, const Path&);
};

} // namespace pe
```

## Behavior & algorithms

- **Storage layout:** a per-kind store on disk (a directory or container) holding
  metadata + thumbnails separately from payloads, so listing a 1000-brush library
  doesn't deserialize every brush.
- **Thumbnails:** generated by rendering the asset (a brush dab stroke, a gradient
  ramp, a swatch chip) at small size and caching.
- **Migration:** each payload type carries a `version`; on load, a migration chain
  upgrades older payloads to the current schema.
- **Import of foreign formats:** format-specific readers convert third-party packs
  into PhotoEdit presets (best-effort, flag unsupported parameters).

## Performance, threading & GPU

- Lazy metadata/thumbnail loading; payloads faulted on use.
- Thumbnail generation runs on [workers](22-performance.md).

## Edge cases & failure modes

- Corrupt/partial pack import → skip bad entries, report a summary, never abort the
  whole import.
- Duplicate ids on import → re-id with provenance kept.
- Missing dependency (a style referencing a missing pattern) → placeholder + warn.
- Version newer than the app → read what's understood, preserve unknown fields.

## Testing strategy

- Unit: add/remove/list/load round-trips; migration from a vN payload to current.
- Import/export a pack and verify byte-stable round-trip for native packs.
- Lazy-load test: listing N presets deserializes 0 payloads.

## Phasing

- **M3**: brush presets + tool presets with the brush engine.
- **M5+**: gradients, patterns, swatches, LUTs as their systems land.
- **M8+**: styles, shapes, Camera Raw presets.
- **M10**: actions, export presets, workspace/shortcut presets, foreign-format
  import, cloud sync, migration tooling.
